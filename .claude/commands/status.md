Run `git status --short` and `git diff --stat` to see what has changed, then summarize it in plain English — no file names, no technical terms.

For example: "You've made changes to the homepage and added a new photo to the gallery. Nothing has been published yet."

If nothing has changed, say: "Everything looks the same as your last save. No changes yet."

After the summary, remind them of their options:
> "When you're ready, you can **/publish** to make your changes live, or **/reset** to undo them."

If git is not set up yet, tell them: "Status tracking isn't available yet — let the site owner know and they can get it ready for you."
